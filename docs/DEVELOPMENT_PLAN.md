# SeaControll 开发计划

## 总目标

构建一个面向 `ESP32 / ESP01S` 的可配置 IoT 控制平台，支持：

- 多账户
- 用户名密码登录
- 登录后绑定 Passkey
- 控制器链接码绑定
- 驱动化设备能力
- GPIO 绑定
- 读写分离的通道模型
- 定时器
- 后续共享驱动市场

## 当前阶段

### 已确认的产品约束

- 平台管理端只做管理员侧能力
- 平台管理端只看用户列表、用户详情、用户设备列表
- 用户端 `pages` 只做普通用户侧能力
- `admin` 与 `pages` 都必须先鉴权，再访问业务数据
- 未鉴权状态只允许显示登录/注册/引导，不允许直接展示列表与配置
- 设备驱动配置、GPIO 配置、定时器配置属于用户侧，不属于平台管理端首页职责

## 近期开发优先级

1. 继续收敛控制器空白入网后的驱动实例流程

3. 补用户设置页（修改密码 / 绑定 Passkey）
4. 细化设备端驱动扩展协议

## 数据存储策略

- 主业务：`PostgreSQL`
- 灵活配置：`JSONB`
- 共享市场：同库分表
- 当前阶段不引入 Redis / 对象存储 / 搜索引擎

## 本地测试方式

- `cd backend && docker compose up --build`
- 后端：`http://localhost:8080`
- 管理端：`http://localhost:8080/admin`
- 数据库：`localhost:5432`
- Adminer：`http://localhost:8081`
- 前端：本地静态服务单独启动
